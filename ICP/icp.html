Presentación de la Asignatura

Introducción

Bienvenido a la asignatura Infraestructuras de Cloud Público (ICP), del Máster Universitario en Computación en la Nube y de Altas Prestaciones (MUCNAP) de la Universitat Politècnica de València. Este asignatura aborda de forma teórica y práctica todos los conceptos, tecnologías y herramientas necesarias para iniciarte en el uso de infraestructuras Cloud como soporte a la ejecución de aplicaciones. Conocerás de primera mano las ventajas que ofrece Cloud Computing y harás prácticas sobre la infraestructura real de Amazon Web Services (AWS), el proveedor líder en Cloud Computing. Trabajarás con los principales servicios de AWS para el aprovisionamiento de infraestructura virtual y para la gestión de datos en la nube. En definitiva, utilizarás las principales herramientas para el despliegue y uso de infraestructuras avanzadas en Cloud. Las clases se imparten mediante una metodología de enseñanza-aprendizaje basada en la lección magistral participativa con el uso de la pregunta como mecanismo para la evocación de conceptos. Los boletines de prácticas incluyen de forma detallada las actividades prácticas a realizar en cada sesión.

Tema 1. Infraestructuras de Cloud Público

Este módulo permite que conozcas el concepto de Cloud Computing (Computación en Nube), así como los principales antecedentes y tecnologías afines que han propiciado su auge. Se abordan las tecnologías, herramientas, proveedores, estándares y paradigmas de computación involucrados en el Cloud. Se estudian algunos casos de éxito de la tecnología y se discuten tanto los principales modelos de servicio (IaaS, PaaS y SaaS) como las ventajas/inconvenientes de la externalización del almacenamiento y/o cómputo a un tercer proveedor.

Este tema permite que conozcas los principales servicios de AWS para el aprovisionamiento de recursos de cómputo y de almacenamiento bajo demanda mediante un modelo de pago por uso. También para el despliegue de bases de datos relacionales y no relacionales (NoSQL) en la nube. Esto te permitirá conocer las ventajas/inconvenientes del modelo de servicio en Cloud Infrastructure as a Service (IaaS).

Tema 3. Gestión de Datos en la Nube con Amazon Web Services

Este tema permite que conozcas las principales aproximaciones de gestión de datos en la nube. Se cubre desde el almacenamiento basado en ficheros sobre proveedores de Cloud público, como Amazon S3 hasta almacenamiento basado en volúmenes orientados a bloques (EBS). Se estudian servicios de gestión de bases de datos relacionales en la nube (como Amazon RDS), así como aproximaciones de tipo NoSQL como es el caso de Amazon DynamoDB.

Tema 4. Arquitecturas de Aplicaciones en el Cloud

Este tema aborda el diseño y las arquitecturas de aplicaciones en el Cloud. Se describen los principales requisitos de diseño de estas aplicaciones y se estudia la forma de conseguir estos requisitos usando los servicios de proveedores de Cloud público como Amazon Web Services (AWS) o Microsoft Azure. Se aborda el uso programático de AWS desde Python con boto. Se realiza una práctica de creación de una arquitectura elástica y escalable basada en una aplicación web (WordPress) con uso de base de datos.

Tema 5. Despliegue y Configuración Automatizada (DevOps)

Este módulo permite que conozcas el concepto de DevOps. Se abordan las tecnologías, herramientas y servicios involucrados. En particular, este tema permite que conozcas algunas de las herramientas existentes para la configuración automática de infraestructuras de cómputo sobre plataformas Cloud. Estas herramientas de DevOps, que permiten la creación de recetas de alto nivel para facilitar la automatización en la configuración de máquinas (virtuales). En concreto se aborda de forma práctica Ansible. También se trabaja con Amazon CloudFormation y Elastic Beanstalk como ejemplos de otros servicios destinados al despliegue automatizado de aplicaciones.

Tema 6. Serverless Computing

Este tema aborda serverless computing como un patrón de diseño para la creación de arquitecturas de aplicaciones sin gestión explícita de servidores. Se trata AWS Lambda como servicio principal para el despliegue de arquitecturas de aplicaciones. Se introduce el modelo de ejecución basado en computación reactiva dirigida por eventos soportado por AWS Lambda y se realizan prácticas utilizando dicho servicio. Para ello, es necesario introducir los servicio CloudTrail, como fuente de eventos y auditoria de las acciones realizadas en AWS, así como el servicio CloudWatch para recibir y encaminar eventos sucedidos en la infraestructura.
